# 농수산물 거래 데이터 전처리 파이프라인

본 프로젝트의 `preprocess.py`는 농수산물 거래 데이터의 노이즈를 필터링하고 분산된 원산지 정보를 통합하며, 모델링을 위한 시간 파생 변수를 생성하여 분석에 용이한 데이터 형태로 가공하는 전처리 모듈입니다.

## 🚀 사용 안내 및 변경 요약

### 사용 방법

`preprocess(df)` 함수 하나만 호출하면 5단계의 모든 전처리 과정이 자동으로 순차 실행됩니다.

```python
import pandas as pd
from preprocess import preprocess

# 1. 원본 데이터 불러오기
df = pd.read_csv('data/train.csv')

# 2. 전처리 파이프라인 실행
df_clean = preprocess(df)
```

### 📊 전처리 파이프라인 주요 단계

전처리 모듈은 데이터의 신뢰성을 높이고 분석 편의성을 극대화하기 위해 아래와 같이 **5개의 세부 단계**로 나뉘어 동작합니다.

#### 1단계: 수치 및 유효성 전처리 (`clean_numeric_columns`, `filter_invalid_rows`)
문자열 형태가 섞인 숫자 데이터의 불순물을 제거하고 무효 데이터를 필터링합니다.
- **콤마 제거 및 형 변환**: `총거래금액`, `총거래물량`, `평균가격` 컬럼에 섞여있는 천 단위 구분 기호(`,`)를 일괄 제거하고 숫자형(`float/int`) 데이터로 안전하게 강제 변환합니다.
- **오류 레코드 삭제**: 비정상적인 거래 내역(환불, 반품, 단순 기입 오류 등)을 제외하기 위해, 총거래물량이 1kg 이상이고 금액과 수량이 0보다 큰 유효한 행정 데이터만 필터링합니다. 

#### 2단계: 산지 및 지역 정보 정제 (`standardize_origin`)
데이터에 서로 다르게 기록된 행정구역 명칭을 최신 기준으로 병합하여 다루기 쉽게 통일합니다.
- **국산/수입 감별**: 특정 국가명 리스트와 키워드('수입산', '(수입)')를 사용해 `국산여부`를 분류합니다.
- **초기화 및 표준화**: '강원도'를 '강원특별자치도'로, '전북'을 '전북특별자치도' 등으로 업데이트된 최신 행정구역 표기법에 맞게 병합합니다.
- **최종 원산지 명칭 작성**: 국내산인 경우 `[광역시도명] + [시군구명]` 결합을 통해 세분화하고, 수입산인 경우 `수입(국가명)` 구조로 통합한 후 최종 시각화에 사용할 수 있는 **`산지`** 단일 컬럼을 도출합니다.

#### 3단계: 시간 데이터 파생 변수 생성 (`_preprocess_date`)
모델이 시계열 트렌드와 계절성을 스스로 학습할 수 있도록 날짜 컬럼을 분할 및 파싱합니다.
- `DATE` 컬럼의 혼재된 날짜 형식을 완벽한 Datetime 객체로 캐스팅합니다.
- 모델링 단계에서 다양한 차원으로 탐색할 수 있도록 `Year`, `Quarter`, `Month`, `Week`(주차), `Day`, `DayOfWeek`(요일) 등의 다양한 시간 피처를 자동으로 생성합니다.

#### 4단계: 핵심 컬럼 선택 (`select_essential_columns`)
분산되어 있는 수많은 열(Column) 중에서 모델링 및 분석에 필요한 핵심 데이터만을 골라내고 순서를 정렬합니다.
- 데이터 퀄리티 강화를 위해 `총거래금액 / 총거래물량`을 바탕으로 `평균가격(kg당)`을 최신화합니다.
- 이후 필수 컬럼만 추출하여, Date > 수치형 데이터 > 범주/분류 피처 > 파생 시계열 피처 등의 직관적인 순서로 데이터프레임을 재단합니다.

#### 5단계: 인적 오류 및 이상치 제거 (`remove_human_errors`)
단순한 형태적 오류가 아닌, 현장 작업자의 입력 실수(Human Error)로 추정되는 파괴적인 이상치를 통계적으로 잘라냅니다.
- **다단 Fallback 중앙값 추출**: `[Year, Week, 품종, 등급]` 그룹 단위로 평균가격의 중앙값(Median)을 계산합니다. 데이터 희소성(Sparsity) 문제가 발생해 중앙값이 구해지지 않을 경우를 대비하여, 상위 그룹인 `[Year, Week, 품종]` ➝ `[Year, Week]` 순으로 백업 중앙값을 산출하는 3단계 방어 로직이 적용되어 있습니다.
- 특정 거래의 평균 가격이 계산된 중앙값 대비 매우 낮거나(0.05배 미만), 아주 높다면(20.0배 초과) 이를 이상치(Outlier)로 판단하고 데이터셋에서 제외시켜 평균값의 왜곡을 방지합니다.
