# 농수산물 거래 데이터 전처리 파이프라인

본 프로젝트의 `preprocess.py`는 농수산물 거래 데이터의 노이즈를 필터링하고, 거래 단위와 원산지(산지) 정보를 표준화하여 분석에 용이한 형태로 가공하는 전처리 모듈입니다.

## 🚀 사용 안내 및 변경 요약

### 사용 방법

`preprocess(df)` 함수 하나만 호출하면 모든 전처리 과정이 자동으로 순차 실행됩니다.

```python
import pandas as pd
from preprocess import preprocess

# 1. 원본 데이터 불러오기
df = pd.read_csv('data/train.csv')

# 2. 전처리 파이프라인 실행
df_clean = preprocess(df)
```

## 📖 상세 설명 (로직 동작 원리)

전처리 파이프라인은 아래와 같이 4개의 세부 모듈로 나누어 데이터를 가공합니다.

### 1. 숫자형 데이터 정제 (`clean_numeric_columns`)
문자열 형태가 섞인 숫자 데이터의 불순물을 제거합니다.
- 데이터에 가끔 포함되어 있는 천 단위 구분 기호 `,` (콤마) 문자를 모두 찾아서 제거합니다.
- 제거된 데이터를 판다스의 숫자형 데이터 형식(`float` 또는 `int`)으로 강제 변환합니다. 처리가 불가능한 심각한 문자열 스팸이 있다면 `NaN`(비어있음) 처리합니다.

### 2. 무효 데이터 필터링 (`filter_invalid_rows`)
통계나 기계학습 모델링에 악영향을 미치는 쓰레기값을 잘라냅니다.
- 환불 처리, 반품, 또는 입력 오류로 추정되는 '금액/물량/단가가 0이거나 음수'인 잘못된 거래 내역을 행 단위로 삭제(Drop)합니다.
- 삭제 전/후의 행 수량과 데이터 유지율을 텍스트로 출력하여 데이터 손실률을 파악할 수 있게 해줍니다.

### 3. 거래 단위의 표준화 (`standardize_unit_capacity`)
단위의 파편화 현상을 해결하고 절대적인 무게 기준을 잡아줍니다.
- 원본 `거래단위` 컬럼은 "1kg상자", "500g망", "5ton트럭" 같이 숫자와 단위가 묶여 있어 직접 계산이 불가능합니다.
- 정규표현식을 통해 숫자 부분(포장 크기)과 텍스트 부분(단위)을 기계적으로 추출합니다.
- 수집된 단위명에서 `g`는 `0.001`배, `ton`이나 `M/T`는 `1000`배 하는 **환산 계수(Factor)**를 부여합니다. `kg`는 기준이므로 따로 건드리지 않습니다.
- 최후 연산인 `포장 크기 * 팩터 * 현 총거래물량`을 수행한 **`총거래물량(kg)`** 컬럼을 새롭게 만듭니다.

### 4. 원산지 명칭 정리 (`standardize_origin`)
지역별 시각화나 수입산과 국산의 가격 비교 등을 할 때 사용하기 위해 원산지 데이터를 명확히 분리하고 통합합니다.
- **국산/수입 감지 (`_classify_origin_type`)**: 
  - 특정 국가 이름('중국', '미국' 등)이나 시군구에 '수입산' 같이 직접적인 키워드를 품고 있거나, 품종 이름에 `(수입)`이란 단어가 있으면 모두 `'수입'` 카테고리로 묶습니다.
  - 국산 데이터로 인지된 경우, 이전 행정 구역의 표기('강원도', '경북' 등)를 최신 주소 규격('강원특별자치도', '경상북도')을 반영하여 통일시킵니다.
- **시군구 오류 정제 (`_clean_sigungu_names`)**: 
  - 엑셀 기입 실수 등으로 시군구 기입란에 '강원특별자치도' 같은 상위 광역시도가 중복 기입되어 있는 쓸모없는 정보를 `기타/미상`으로 필터링 처리합니다.
- **최종 '산지' 조합 (`_format_final_origin`)**: 
  - 국내산일 경우 `[광역시도명] + [시군구명]` 결합을 통해 `경기도 안성시` 와 같이 만듭니다.
  - 수입산일 경우 `[광역시도명]` 위치에 있는 실제 국가명을 사용하여 `수입(미국)` 형태로 뼈대를 잡습니다.
  - 최종 도출된 이 **`산지`** 컬럼 하나만 그룹바이(GroupBy)하면 모든 위치별 통계를 쉽게 확인할 수 있습니다.
